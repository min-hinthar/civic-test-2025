<task type="auto">
  <name>Task 2: Add micro-interactions to components identified as MISSING</name>
  <files>Components identified as MISSING in the audit (estimated 4-8 files)</files>
  <action>
For each component identified as MISSING micro-interactions in the audit:

**Pattern for adding whileTap feedback:**

If the element is already a `motion.*` element:
```tsx
<motion.button
  whileTap={{ scale: 0.97 }}
  transition={SPRING_SNAPPY}
  // ... existing props
>
```

If the element is a plain HTML element, convert to motion:
```tsx
import { motion } from 'motion/react';
import { SPRING_SNAPPY } from '@/lib/motion-config';

// Before:
<button onClick={...}>

// After:
<motion.button
  onClick={...}
  whileTap={{ scale: 0.97 }}
  transition={SPRING_SNAPPY}
>
```

**Feedback tier mapping (from CONTEXT.md decisions):**
- Primary buttons (main CTA): `whileTap={{ scale: 0.95 }}` with `SPRING_BOUNCY`
- Secondary buttons (filter chips, options): `whileTap={{ scale: 0.97 }}` with `SPRING_SNAPPY`
- Tertiary/subtle (text links, small icons): `whileTap={{ opacity: 0.7 }}` with `SPRING_SNAPPY`
- Cards: `whileHover={{ scale: 1.02 }}` with `SPRING_GENTLE` (if not already present)

Import `SPRING_BOUNCY`, `SPRING_SNAPPY`, or `SPRING_GENTLE` from `@/lib/motion-config`.

Do NOT add micro-interactions to:
- Elements that already have whileTap/whileHover
- Non-interactive elements (display-only text, images)
- Elements where the parent handles the interaction (e.g., a span inside a motion.button)
- Disabled elements
