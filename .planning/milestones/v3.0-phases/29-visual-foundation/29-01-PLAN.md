<task type="auto">
  <name>Task 1: Add semantic fontSize entries to tailwind.config.js</name>
  <files>tailwind.config.js</files>
  <action>
Add a `fontSize` section to `theme.extend` in `tailwind.config.js` with the following semantic scale entries:

```javascript
fontSize: {
  'caption': ['0.625rem', { lineHeight: '0.875rem', letterSpacing: '0.01em' }],
  'body-xs': ['0.75rem', { lineHeight: '1rem' }],
  'body-sm': ['0.875rem', { lineHeight: '1.25rem' }],
  'body': ['1rem', { lineHeight: '1.5rem' }],
  'body-lg': ['1.125rem', { lineHeight: '1.75rem' }],
  'heading-sm': ['1.25rem', { lineHeight: '1.75rem', fontWeight: '600' }],
  'heading': ['1.75rem', { lineHeight: '2.25rem', fontWeight: '700' }],
  'display': ['2.25rem', { lineHeight: '2.75rem', fontWeight: '800' }],
},
```

Place inside `theme.extend` alongside the existing `colors`, `borderRadius`, etc. sections.

IMPORTANT: Use `theme.extend.fontSize` (NOT `theme.fontSize`) to preserve all default Tailwind text sizes. Overriding would break 1,084+ existing text-size usages.

Reference: Tailwind v3 docs — fontSize configuration supports `[size, { lineHeight, letterSpacing, fontWeight }]` tuple syntax.
  </action>
  <verify>
Run `npm run build` -- must succeed with no errors.
Grep the tailwind config for `caption` to confirm the new entries exist.
Run `npx tailwindcss --content src/styles/tokens.css --output /dev/null 2>&1` or simply verify the dev server starts cleanly with `timeout 10 npm run dev` (kill after confirming no errors).
  </verify>
