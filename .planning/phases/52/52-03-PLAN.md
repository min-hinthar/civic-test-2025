IMPORTANT per D-08: The session-lock deferral test is the KEY test for this spec — it validates Phase 50's SW update deferral behavior. The mock must simulate both `updatefound` on registration and `controllerchange` on `navigator.serviceWorker`.

SW mock approach (from research Pattern 4):
```typescript
await page.evaluate(() => {
  // Create mock waiting worker
  const mockWaiting = { state: 'installed', postMessage: () => {} };

  // Store existing registration or create mock
  const mockReg = {
    installing: null,
    waiting: mockWaiting,
    active: { state: 'activated' },
    addEventListener: (event, handler) => {
      if (event === 'updatefound') handler();
    },
    removeEventListener: () => {},
  };

  // Override getRegistration
  if (navigator.serviceWorker) {
    Object.defineProperty(navigator.serviceWorker, 'getRegistration', {
      value: () => Promise.resolve(mockReg),
      writable: true,
    });
    // Fire controllerchange
    navigator.serviceWorker.dispatchEvent(new Event('controllerchange'));
  }
});
```
