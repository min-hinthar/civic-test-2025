## Component Visual Contracts

### SharedErrorFallback (new shared presentational component)

```
+---------------------------------------------+
|           [amber circle bg]                  |  h-16 w-16 rounded-full bg-amber-100
|           [AlertCircle icon]                 |  h-8 w-8 text-warning
|                mb-6                          |
|    "Something went wrong"                    |  text-xl font-semibold text-foreground
|    "..." (Burmese)                           |  text-lg font-myanmar text-foreground mt-1
|                mb-8                          |
|    [Try again]      [Return home]            |  min-h-[44px] gap-3
+---------------------------------------------+
  Container: max-w-md, p-8, rounded-2xl, bg-muted/30
  Outer: min-h-[400px] flex items-center justify-center p-6
```

**Props:**
- `message: BilingualMessage` -- sanitized error text
- `showBurmese: boolean` -- from useLanguage() or localStorage fallback
- `onRetry?: () => void` -- "Try again" button handler
- `onGoHome?: () => void` -- "Return home" button handler

**Used by:** `ErrorBoundary.tsx` (class, localStorage lang), `app/error.tsx` (functional, useLanguage hook), `app/not-found.tsx`

**Touch targets:** All buttons `min-h-[44px] min-w-[44px]` -- mandatory per CLAUDE.md mobile-first PWA requirement.

**Glass tier:** `bg-muted/30` (glass-light equivalent for error cards). Full-page error overlays do NOT use backdrop-filter -- error pages must render even when CSS is partially broken.

**Responsive:** Buttons stack vertically on mobile (`flex-col`), side-by-side on sm+ (`sm:flex-row sm:justify-center`).

### app/error.tsx (route error page)

- Calls `sanitizeError(error)` before rendering -- never displays raw `error.message`
- Uses `useLanguage()` hook for language detection (IS inside provider tree)
- Falls back to localStorage `civic-test-language-mode` if hook fails
- Renders SharedErrorFallback with both "Try again" (calls `reset()`) and "Return home"
- Reports to Sentry via `captureException` in useEffect

### app/global-error.tsx (catastrophic fallback)

- Minimal HTML -- no Tailwind classes (CSS may be broken)
- Inline styles only: `padding: 2rem`, `textAlign: center`, `fontFamily: system-ui`
- Shows both English + Burmese text (hardcoded, no hook/localStorage dependency)
- "Try again" button only (no router available)
- Colors: CSS custom properties with inline fallbacks (`color: hsl(222, 47%, 12%)`)

### app/not-found.tsx (404 page)

- Large "404" display number: `text-display font-extrabold text-primary`
- Bilingual heading: "Page Not Found" / Burmese equivalent
- Bilingual body text with next-step guidance
- "Return home" Link component: `min-h-[44px]` button styling
- Uses `useLanguage()` hook (inside provider tree)

### Session Error Boundary Fallback (component-level)

- Same SharedErrorFallback visual, but:
  - "Try again" resets the error boundary (re-mounts session component)
  - "Return home" navigates to `/`
  - Error boundary triggers cleanup BEFORE rendering fallback:
    - InterviewSession: cancel TTS, 3 audio players, speech recognition, recording, 2 timeout refs
    - PracticeSession: clear checkDelayRef, streakTimerRef
    - TestPage: `setLock(false)` (CRITICAL -- releases nav lock), clear timeout refs
    - CelebrationOverlay: `fallback={null}` (silent failure, post-session)

### Copy Voice Rules

- **Warm Coach tone**: "Don't worry" not "An error has occurred"
- **Solution-focused**: Lead with what to do, not what broke
- **No blame**: "Connection interrupted" not "You closed the app"
- **No jargon**: Never show SQL, database, authentication token, stack trace
- **No ALL CAPS**: Never FATAL, CRASH, FAILED
- **No raw error codes**: Users never see error.message directly
- **Reassurance over precision**: "Your progress has been saved" matters more than technical accuracy

Source: `49-PRECONTEXT-RESEARCH.md` section 8, `49-CONTEXT.md` decisions

## Interaction Contracts

### Error Recovery Flow

1. Error thrown in component tree
2. Nearest ErrorBoundary catches it
3. `componentDidCatch` fires:
   a. Cleanup resources (cancel TTS/audio/timers, release nav lock)
   b. Report to Sentry (sanitized, PII-stripped)
   c. Call `onError` callback if provided
4. `getDerivedStateFromError` sets `hasError: true` with sanitized message
5. SharedErrorFallback renders with two options:
   - "Try again" -- resets boundary, re-mounts child tree
   - "Return home" -- `window.location.href = '/'` (hard navigation, cleans all state)

### Error Propagation Cascade

```
Component throw
  -> Session ErrorBoundary (cleanup + localized fallback)
      -> if not caught:
  -> Page ErrorBoundary (bilingual fallback)
      -> if not caught:
  -> app/error.tsx (sanitized + bilingual)
      -> if not caught:
  -> app/global-error.tsx (minimal HTML, inline styles, hardcoded bilingual)
```

### Provider Ordering Guard (dev-mode only)

- `<ProviderOrderGuard />` renders as null component at bottom of provider tree
- Calls every context hook in try-catch
- Violations produce `console.warn` with correct ordering reference
- Zero production overhead (`process.env.NODE_ENV === 'development'` guard)
- Does NOT crash the app -- warning only

### Session Auto-Save Reliance

- All session components auto-save every 5 seconds to IndexedDB
- Error boundary does NOT re-save -- snapshot is max 5 seconds old
- This is a deliberate decision to avoid race conditions in error handlers

Source: `49-CONTEXT.md` decisions, `49-PRECONTEXT-RESEARCH.md` sections 5, 9

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | All error buttons: `min-h-[44px] min-w-[44px]` |
